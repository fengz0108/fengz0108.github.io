Git 学习笔记

什么是Git

Git 是世界上最先进的版本控制器（没有之一）。是Linux的创始人Linus为了方便Linux的代码花费两周而开发出来的一套分布式版本控制器。

突然不禁感叹牛人就是分分钟改变世界！

安装Git

在Windows下安装

只需下载安装Git客户端默认安装就好了。

安装完成之后在开始菜单里找到Git Bash，通过这个使用Git。然后进行进一步设置，在命令行里输入：

$ git config --global user.name "Your Name"
$ git config --global user.email "email@example.com"

注意git config命令的--global参数，用了这个参数，表示你这台机器上所有的Git仓库都会使用这个配置，当然也可以对某个仓库指定不同的用户名和Email地址。

在Linux下安装

首先，试着输入git看系统是否已经安装Git：

$ git
The program 'git' is currently not installed. You can install it by typing:
sudo apt-get install git

若无，输入sudo apt-get install git安装即可

创建版本库

什么是版本库呢？版本库又名仓库，英文名repository，你可以简单理解成一个目录，这个目录里面的所有文件都可以被Git管理起来，每个文件的修改、删除，Git都能跟踪，以便任何时刻都可以追踪历史，或者在将来某个时刻可以“还原”。

创建方法

第一步，创建一个新目录：

$ mkdir learngit
$ cd learngit
$ pwd
/Users/michael/learngit

pwd命令是显示当前目录。

第二步，初始化仓库：

$ git init

添加文件到仓库

在仓库目录下新建文件，然后使用命令

$ git add <file>
$ git commit -m "说明与注释"

可以add很多文件，一次性commit，此外查看工作区状态使用git status命令

若git status告知有文件改动，可以使用git diff查看修改内容

版本退回

  • HEAD指向当前版本，使用git reset --hard commit_id可以在历史版本之前穿梭
  • 可以使用git log来查看历史版本以及他们的commit_id
  • 要重返未来，可以使用git relog来查看历史版本。

撤消修改

  • 若是你乱改了工作区的内容，想丢工作区的修改只需使用命令git checkout -- file
  • 若是你不进修改了工作区还add了那就先git reset HEAD file,然后再按上一条恢复
  • 若是已经commit则先版本退回

删除文件

$ git rm <file>

误删还原

$ git checkout -- <file>

远程仓库

什么是Github

Github是一个提供Git仓库托管的网站，只需注册一个Github账号就能免费获得Git的远程仓库。

由于你的本地Git仓库和GitHub仓库之间的传输是通过SSH加密的，所以，需要一点设置：

第一步：创建SSH Key。在Windows下打开Git Bash创建：

$ ssh -keygen -t rsa -C "youremail@example.com"

无需设置密码一路回车即可。然后你会在用户目录里找到.ssh目录，里面有id_rsa和id_rsa.pub两个文件，这两个就是SSH Key的密钥对，is_rsa是私钥不可泄露，id_rsa.pub是公钥，可以告诉别人。

第二步：登录Github，打开”Account settings”,”SSH Key”页面，点击”Add SSH Key”,填上任意Title将id_rsa.pub里的内容复制进去即可。

添加远程仓库

首先登录Github创建一个仓库

创建本地仓库与远程仓库的关联，在本地仓库下运行命令：

$ git remote add origin git@github.com:fengz0108/xxx.git

本地内容推送到远程：

$ git push -u origin master

由于远程库是空的，我们第一次推送master分支时，加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

$ git push origin master

远程克隆

$ git clone git@github.com:fengz0108/xxx.git
